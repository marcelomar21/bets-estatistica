# 4.3 Integrar tom de voz no copyService

## Objetivo

Modificar o copyService para aceitar configuracao de tom de voz per-group, migrando de raw string prompt para ChatPromptTemplate com system message.

## Action

1. Modificar `generateBetCopy(bet)` -> `generateBetCopy(bet, toneConfig)`
2. Migrar de raw string prompt para `ChatPromptTemplate.fromMessages([['system', systemMsg], ['human', humanMsg]])`
3. System message injetado com: persona, tom, palavras proibidas, regras customizadas do grupo
4. Invalidar cache quando `copy_tone_config` muda (via timestamp de ultima atualizacao)

## Files

| Arquivo | Descricao |
|---------|-----------|
| `bot/services/copyService.js` | Prompt fixo -> tone config per-group via ChatPromptTemplate |

## Acceptance Criteria

- [ ] `generateBetCopy(bet, toneConfig)` aceita config de tom e gera copy respeitando as regras
- [ ] System message inclui persona, tom, palavras proibidas e regras customizadas
- [ ] Cache e invalidado quando `copy_tone_config` muda

## Notes

- O `toneConfig` e carregado via `BotContext.groupConfig.copyToneConfig` (Fase 2). Ja usa a abstracao correta.
- `copyService.js` usa `config.llm.lightModel` (atualmente `gpt-5-mini`) com `temperature: 0.2`, `maxTokens: 200`.
- Prompt e raw string (sem `ChatPromptTemplate`, sem system message).
- Regras incluem "Abrevie nomes de times" com 1 exemplo — insuficiente para traducao.
- Nomes de times vem da FootyStats API as-is (ingles para ligas europeias).
- **Nao existe nenhuma camada de traducao** de nomes de times.
- Cache in-memory com TTL de 24h e max 200 entries (limpa no restart).
