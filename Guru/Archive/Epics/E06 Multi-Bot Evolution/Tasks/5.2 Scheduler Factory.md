# 5.2 Refatorar scheduler para factory pattern

## Objetivo

Transformar o scheduler de module globals em uma factory function que cria uma instancia isolada por grupo, garantindo isolamento de falhas.

## Action

1. Exportar `createScheduler(groupId, botCtx)` factory function
2. Cada instancia tem seu proprio: `activePostingJobs`, `currentSchedule`, `isManualPostInProgress`
3. Em `server.js`: criar 1 scheduler por grupo ativo
4. `reloadPostingSchedule()` e `checkPostNow()` sao per-scheduler instance
5. Intervalos de reload/polling: 1 `setInterval` global que itera sobre todos os schedulers (evita N timers)

## Files

| Arquivo | Descricao |
|---------|-----------|
| `bot/server.scheduler.js` | Module globals -> factory per group com isolamento |

## Acceptance Criteria

- [ ] **AC 5.2**: Given posting schedule de Guru as 10h e Osmar as 10:05, when ambos os horarios chegam, then cada bot posta no seu grupo correto sem interferencia.
- [ ] **AC 5.3**: Given o handler de Osmar lanca uma excecao, when o erro e capturado, then o bot do Guru continua funcionando normalmente (isolamento de falhas).

## Notes

- Isolamento de falha: se scheduler de grupo A falha, grupo B continua.
- Estado atual: `let activePostingJobs = []`, `let currentSchedule = null`, `let isManualPostInProgress = false` — globals compartilhados.
- `loadPostingSchedule()` le `groups.posting_schedule` filtrado por `GROUP_ID`.
- `setupDynamicScheduler(schedule)` cria pares de cron jobs por horario: distribuicao em `T-5min`, postagem em `T`.
- `reloadPostingSchedule()` a cada 5min via `setInterval` detecta mudancas no DB.
- `checkPostNow()` a cada 30s via `setInterval` detecta flag `post_now_requested_at`.
