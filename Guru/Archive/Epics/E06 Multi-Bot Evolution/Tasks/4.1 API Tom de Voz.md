# 4.1 API endpoint para tom de voz

## Objetivo

Criar endpoints REST para leitura e atualizacao da configuracao de tom de voz per-group, incluindo conversao de texto livre em config estruturada via LLM.

## Action

1. `GET /api/groups/[id]/tone` ‚Äî retorna `copy_tone_config` do grupo
2. `PUT /api/groups/[id]/tone` ‚Äî atualiza `copy_tone_config` (validacao via Zod)
3. Acesso: `super_admin` pode editar qualquer grupo, **`group_admin` pode editar seu proprio grupo**

**Conversao texto livre -> config estruturada (Nivel 1 -> JSONB)**:
- Modelo: `gpt-5-mini` (config.llm.lightModel ‚Äî rapido, barato, bom para extracao estruturada)
- Prompt: system message com schema esperado + exemplos few-shot de conversao
- Schema de saida (Zod `withStructuredOutput`):
  ```js
  z.object({
    persona: z.string().optional(),
    tone: z.string(),
    forbiddenWords: z.array(z.string()),
    ctaText: z.string().optional(),
    customRules: z.array(z.string()),
    rawDescription: z.string() // preserva o texto original
  })
  ```
- **Error handling**: se LLM falha ou retorna schema invalido, salvar apenas `{ rawDescription: textoOriginal }` e notificar admin que a conversao falhou. O copyService usa `rawDescription` como fallback no system prompt.
- **Validacao**: rejeitar se `forbiddenWords` contem >50 itens ou se `customRules` contem >20 regras (limites sensatos).
