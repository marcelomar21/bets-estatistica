# 5.4 Propagar groupId para todos os jobs

## Objetivo

Cada job recebe `(groupId, botCtx)` em vez de ler `config.membership.groupId`, garantindo isolamento per-group.

## Action

1. Cada job recebe `(groupId, botCtx)` em vez de ler `config.membership.groupId`
2. `pendingConfirmations` em `postBets.js` vira scoped: `Map<groupId, Map<confirmId, ...>>`
3. Jobs centrais (distributeBets, trackResults) continuam rodando 1x cross-group, mas passam o `botCtx` correto ao enviar alertas

Jobs a modificar:
- `bot/jobs/postBets.js` — pendingConfirmations Map -> scoped per-group
- `bot/jobs/distributeBets.js` — passa botCtx ao enviar alertas
- `bot/jobs/trackResults.js` — passa botCtx ao enviar alertas
- `bot/jobs/membership/kick-expired.js` — recebe groupId param
- `bot/jobs/membership/sync-group-members.js` — recebe groupId param
- `bot/jobs/membership/renewal-reminders.js` — recebe groupId param

## Files

| Arquivo | Descricao |
|---------|-----------|
| `bot/jobs/postBets.js` | pendingConfirmations Map global -> scoped per-group |
| `bot/jobs/distributeBets.js` | Passa botCtx ao enviar alertas |
| `bot/jobs/trackResults.js` | Passa botCtx ao enviar alertas |
| `bot/jobs/membership/kick-expired.js` | Recebe groupId param em vez de config global |
| `bot/jobs/membership/sync-group-members.js` | Recebe groupId param em vez de GROUP_ID global |
| `bot/jobs/membership/renewal-reminders.js` | Recebe groupId param em vez de GROUP_ID global |

## Acceptance Criteria

- [ ] Nenhum job le `config.membership.groupId` diretamente
- [ ] `pendingConfirmations` e scoped per-group
- [ ] Jobs de membership recebem groupId como parametro

## Notes

- `pendingConfirmations` Map e global em `postBets.js` — precisa scope per-group para evitar colisao de IDs.
- `BOT_MODE`:
  - `central`: jobs globais (distributeBets, trackResults, enrichOdds, kick-expired)
  - `group`: jobs per-group (posting scheduler, renewalReminders, syncMembers)
  - `mixed`: todos os jobs (default)
  - Central jobs NAO usam `GROUP_ID` — rodam cross-group
- Jobs de membership usam `GROUP_ID` global atualmente.
