<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| group_admin -> checkout API | group_admin can only checkout for their own group |
| checkout API -> Mercado Pago | External API call with financial implications |
| client browser -> MP checkout | Payment happens on Mercado Pago's domain (trusted) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Elevation of Privilege | checkout API | mitigate | groupFilter check ensures group_admin can only checkout for own group |
| T-03-09 | Tampering | price manipulation | mitigate | Price calculated server-side from league_pricing and league_discounts tables; client cannot specify price |
| T-03-10 | Spoofing | fake checkout completion | mitigate | Status transitions (pending -> active) only happen via webhook processing (Plan 04), not via client API |
| T-03-11 | Repudiation | checkout transactions | mitigate | group_league_subscriptions records created with timestamps; MP plan ID stored for audit trail |
| T-03-12 | Denial of Service | repeated checkout calls | accept | group_admin is authenticated user; upsert prevents duplicate subscription records |
</threat_model>
