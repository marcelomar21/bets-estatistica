    ```sql
    ALTER TABLE league_seasons ADD COLUMN IF NOT EXISTS tier TEXT NOT NULL DEFAULT 'standard';
    ALTER TABLE league_seasons ADD CONSTRAINT chk_league_seasons_tier CHECK (tier IN ('standard', 'extra'));
    ```

    **2. Create `league_pricing` table** (stores per-league pricing, one row per league_name):
    ```sql
    CREATE TABLE IF NOT EXISTS league_pricing (
      id SERIAL PRIMARY KEY,
      league_name TEXT NOT NULL UNIQUE,
      monthly_price NUMERIC(10,2) NOT NULL DEFAULT 200.00,
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
    );
    ```
    Add index: `CREATE INDEX idx_league_pricing_name ON league_pricing(league_name);`

    **3. Create `group_league_subscriptions` table** (tracks which group has active subscription to which extra league):
    ```sql
    CREATE TABLE IF NOT EXISTS group_league_subscriptions (
      id SERIAL PRIMARY KEY,
      group_id UUID NOT NULL REFERENCES groups(id) ON DELETE CASCADE,
      league_name TEXT NOT NULL,
      status TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'active', 'cancelled', 'expired')),
      mp_plan_id TEXT,
      mp_checkout_url TEXT,
      activated_at TIMESTAMPTZ,
      cancelled_at TIMESTAMPTZ,
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      UNIQUE(group_id, league_name)
    );
    ```
    Add indexes:
    - `CREATE INDEX idx_gls_group ON group_league_subscriptions(group_id);`
    - `CREATE INDEX idx_gls_status ON group_league_subscriptions(group_id, status);`
    - `CREATE INDEX idx_gls_mp_plan ON group_league_subscriptions(mp_plan_id) WHERE mp_plan_id IS NOT NULL;`

    **4. Create `league_discounts` table** (percentage discount per group per league):
    ```sql
    CREATE TABLE IF NOT EXISTS league_discounts (
      id SERIAL PRIMARY KEY,
      group_id UUID NOT NULL REFERENCES groups(id) ON DELETE CASCADE,
      league_name TEXT NOT NULL,
      discount_percent INTEGER NOT NULL CHECK (discount_percent > 0 AND discount_percent <= 100),
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      UNIQUE(group_id, league_name)
    );
    ```
    Add index: `CREATE INDEX idx_league_discounts_group ON league_discounts(group_id);`

    **5. RLS policies for all new tables** (follow same pattern as migration 049):

    For `league_pricing`:
    - Enable RLS
    - super_admin: full access (SELECT, INSERT, UPDATE, DELETE)
    - group_admin: SELECT only (read prices, cannot modify)

    For `group_league_subscriptions`:
    - Enable RLS
    - super_admin: full access
    - group_admin: own group only (USING `group_id = group_league_subscriptions.group_id`)

    For `league_discounts`:
    - Enable RLS
    - super_admin: full access
    - group_admin: SELECT own group only (can see their discounts, cannot modify)

    Each RLS policy must use the established pattern:
    ```sql
    CREATE POLICY {name} ON {table}
      FOR ALL TO authenticated
      USING (
        EXISTS (SELECT 1 FROM admin_users WHERE id = auth.uid() AND role = 'super_admin')
      )
      WITH CHECK (
        EXISTS (SELECT 1 FROM admin_users WHERE id = auth.uid() AND role = 'super_admin')
      );
    ```

| Boundary | Description |
|----------|-------------|
| admin panel API -> Supabase | Tier/pricing/discount mutations require super_admin role |
| group_admin -> league_discounts | group_admin can only read discounts for their own group |
| SQL migration -> production DB | Schema changes are irreversible after data is written |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | tiers/pricing/discounts APIs | mitigate | All mutation endpoints restricted to super_admin via createApiHandler allowedRoles |
| T-03-02 | Tampering | league_pricing.monthly_price | mitigate | Zod validation ensures monthly_price > 0; CHECK constraint on discount_percent 1-100 |
| T-03-03 | Information Disclosure | league_discounts | mitigate | RLS policy ensures group_admin can only SELECT own group discounts |
| T-03-04 | Denial of Service | bulk tier update | accept | Super admin is trusted user; no rate limiting needed for admin-only endpoints |
</threat_model>
